# 🧪 كيفية اختبار الانيمشنات السلسة

## ✅ خطوات الاختبار

### 1️⃣ تشغيل التطبيق

```bash
cd e:\app_transport\app_transport
flutter run
```

أو للويب:

```bash
flutter run -d web
```

---

### 2️⃣ اختبار الصفحات الرئيسية

#### أولاً: اختبر Home Page → Flying Taxi Detail

1. افتح التطبيق
2. انقر على أي رحلة Flying Taxi في القائمة
3. **لاحظ:** انيمشن slideRight من اليمين ✅

#### ثانياً: اختبر Home Page → Transit Detail

1. عد للصفحة الرئيسية
2. انقر على أي رحلة Transit في القائمة
3. **لاحظ:** نفس الانيمشن slideRight ✅

#### ثالثاً: اختبر Transit Trips → ChatBot

1. اذهب إلى Transit Trips
2. انقر على AI Banner (الإقتراح AI)
3. **لاحظ:** انيمشن zoomIn (تكبير) ✅

#### رابعاً: اختبر My Bookings → Detail

1. اذهب إلى My Bookings
2. انقر على أي حجز
3. **لاحظ:** انيمشن slideUp من الأسفل ✅

#### خامساً: اختبر ChatBot Fullpage

1. من أي صفحة، افتح ChatBot
2. **لاحظ:** انيمشن slideUp من الأسفل ✅

---

### 3️⃣ ما الذي يجب أن تلاحظه

#### ✨ العلامات الإيجابية (الانيمشن صحيح):

- [ ] **السلاسة:** لا توجد جزفات jumps أو تأخير
- [ ] **المدة المناسبة:** الانيمشن سريع لكن واضح (400-450ms)
- [ ] **الـ Fade:** الشفافية تتغير بسلاسة
- [ ] **الاتجاه:** الانيمشن من الاتجاه المتوقع
- [ ] **التوحيد:** نفس الانيمشن في كل صفحة

#### ❌ العلامات السلبية (إذا حدثت، يوجد مشكلة):

- ❌ الصفحة تظهر مباشرة بدون انيمشن
- ❌ الانيمشن بطيء جداً أو سريع جداً
- ❌ Freeze أو تأخير أثناء الانيمشن
- ❌ الانيمشن جاف أو بدون Fade

---

### 4️⃣ اختبار الأداء

#### قياس الأداء:

```bash
# تشغيل مع تتبع الأداء
flutter run --profile
```

**المتوقع:**

- FPS: 60 fps ثابتة
- بدون dropped frames خلال الانيمشن
- استهلاك الموارد: منخفض

## 🔍 نقاط الفحص المهمة

### قائمة الفحص:

- [ ] لا توجد رسائل خطأ (errors) في console
- [ ] لا توجد رسائل تحذير (warnings) متعلقة بـ animations
- [ ] جميع الصفحات تنتقل بسلاسة
- [ ] الـ back button يعمل بسلاسة
- [ ] الفيديو سلس (60 fps)
- [ ] بدون freeze أو lag
- [ ] الألوان والشفافية تتغير بسلاسة
